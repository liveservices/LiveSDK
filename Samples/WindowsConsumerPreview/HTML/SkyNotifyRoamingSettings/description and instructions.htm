*********SkyDrive Notifications Sample************

Technologies:  JS, HTML5, Windows Runtime

Last updated: 03/01/2012

Operating System Requirements: 

                Client: Windows Consumer Preview + Visual Studio 2011 Consumer Preview

                Server: Visual Studio 2010 + IIS7

This sample demonstrates how to use Sky Drive + Windows Push Notification Services and Roaming Settings.

The sample is integrated by 2 solutions:

1.       SkyNotifyRoamSample.sln  metro application (Client)

2.       WNSSkyDriveService.sln Service that connects to Windows Notification Server for sending notifications between clients. (Server)

In order to run the sample you will need to update some information in each solution with your own data:

1.       First you need to change your package name and your certificate for run the metro app, for that Open Visual Studio 2011 Consumer preview ON Windows 8 Consumer Preview and open the Client side solution (SkyNotifyRoamSample.sln), then go to package.AppxManifest of the solution and change the following information with your own info.

Note: in order to change the publisher name, you need to delete first the key SkyNotifyRoamSample_TemporaryKey.pfx and the click the Choose Certificate button and create a new test certificate.

 


2.       After you have changed that information you will need to go to https://manage.dev.live.com/build?wa=wsignin1.0 and register your metro app, you will obtain these values when done:

·         Package name

·         Client secret

·         Package Security Identifier (SID)

 

3.       Open Visual Studio 2011 Consumer preview and open the Client side solution (SkyNotifyRoamSample.sln), then update AppxManifest of with the new data obtained.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 


4.       Run Visual Studio 2010 with elevated privileges (WNSSkyDriveService.sln)   and update inside the file WNSMethods.cs the correct value for the following variables: Sid and Secret

 

 

5.       The solution already contains an XML as the DataBase where it will store the information about all the clients that are connecting to the service, the file is under the solution folder DB.

 

6.      

You need to modify the variable referencing the path of this file, for that open WNS_DAO.cs class and change the value of the XMLPath variable with the proper path where the xml Database file will be located.


 

7.       In order to deploy the solution, you need to ensure that some features are turned on or your PC, for that:

a.       Go to Start > Control Panel > Programs > Programs and Features \ Turn Windows Features on or off

Check off:

·         Internet Information Services

·         Internet Information Services \ Web Management Tools \ IIS 6 Management Compatibility

o   IIS Metabase and IIS 6 configuration compatibility

·         Internet Information Services \ World Wide Web Services \ Application Development Features:

o   .NET Extensibility

o   ASP

o   ASP.NET

o   ISAPI Extensions

o   ISAPI Filters

 

b.      Run cmd.exe as an administrator and execute the following command: %windir%\Microsoft.NET\Framework64\v4.0.30319\aspnet_regiis i

 

c.       Ensure that Windows Firewall allows incoming HTTP traffic:

·         Go to Start > Control Panel > System and Security > Windows Firewall > Advanced Settings

·         Under Inbound Rules, select World Wide Web Services (HTTP Traffic-In) and select Enable Rule in the Actions pane.

 

8.       After deploying the server (WNSSkyDriveService.sln) go to your client solution (SkyNotifyRoamSample.sln) and point it to the right service url.

-          Go to the file groupedItemsPage.js and update the global variable AppServiceName with the URL of your server

-          Make sure the path resulted for making the call to your web service meets your real path, if not change whats needed when assigning a value to the baseURL variable on the file groupedItemsPage.js

 

9.       After editing these values you should be able to run the sample properly.
